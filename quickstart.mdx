This guide will walk you through setting up the Gist Web SDK and displaying your first in-app message.

## Step 1: Install the SDK

First, install the SDK using your preferred package manager:

<CodeGroup>
```bash npm
npm install customerio-gist-web
```

```bash yarn
yarn add customerio-gist-web
```
</CodeGroup>

## Step 2: Initialize the SDK

Import and initialize the SDK with your Customer.io configuration. This should happen early in your application lifecycle:

```javascript
import Gist from 'customerio-gist-web';

// Initialize with your Customer.io site ID
await Gist.setup({
  siteId: 'your-site-id',        // Required: Your Customer.io site ID
  dataCenter: 'us',               // Required: 'us' or 'eu'
  logging: true,                  // Optional: Enable debug logging
  useAnonymousSession: true       // Optional: Support anonymous users
});
```

<Note>
  The `setup()` method is asynchronous. Make sure to await it before using other SDK methods.
</Note>

## Step 3: Identify Users

For logged-in users, set a user token to identify them:

```javascript
// When user logs in
await Gist.setUserToken('user-unique-id');

// Optionally, set an expiry date for the token
const expiryDate = new Date('2024-12-31');
await Gist.setUserToken('user-unique-id', expiryDate);
```

For anonymous users, the SDK automatically handles session management when `useAnonymousSession` is enabled.

<Tip>
  When a user logs out, call `Gist.clearUserToken()` to clear their session. If `useAnonymousSession` is enabled, a new anonymous session will be created automatically.
</Tip>

## Step 4: Set Current Route

If you're building a single-page application (SPA), set the current route to enable route-based message targeting:

```javascript
// Set initial route
await Gist.setCurrentRoute('home');

// Update route on navigation
await Gist.setCurrentRoute('dashboard');
await Gist.setCurrentRoute('settings');
```

This allows you to target messages to specific pages or sections of your app.

## Step 5: Display Messages

Messages can be displayed automatically based on your Customer.io campaigns, or you can show them programmatically:

<Tabs>
  <Tab title="Modal Message">
    Display a centered modal message:

    ```javascript
    await Gist.showMessage({
      messageId: 'welcome-message',
      position: 'center'
    });
    ```
  </Tab>

  <Tab title="Message with Properties">
    Pass custom properties to personalize message content:

    ```javascript
    await Gist.showMessage({
      messageId: 'artists',
      position: 'center',
      properties: {
        gist: { scale: true },
        title: 'Top Artists',
        list: [
          { 
            name: 'Beatles', 
            discography: ['Please Please Me', 'With The Beatles'] 
          },
          { 
            name: 'The Doors', 
            discography: ['The Doors', 'Strange Days'] 
          }
        ]
      }
    });
    ```
  </Tab>

  <Tab title="Embedded Banner">
    Embed a message in a specific HTML element:

    ```html
    <div id="banner"></div>
    ```

    ```javascript
    await Gist.embedMessage(
      { messageId: 'example-banner' },
      'banner'
    );
    ```
  </Tab>

  <Tab title="Floating Message">
    Embed a message in a preset Gist position:

    ```javascript
    // Available positions:
    // x-gist-top, x-gist-bottom,
    // x-gist-floating-top, x-gist-floating-bottom,
    // x-gist-floating-top-left, x-gist-floating-top-right,
    // x-gist-floating-bottom-left, x-gist-floating-bottom-right
    
    await Gist.embedMessage(
      { messageId: 'example-notice' },
      'x-gist-floating-bottom-right'
    );
    ```
  </Tab>
</Tabs>

## Step 6: Listen to Events

Subscribe to SDK events to track message interactions and respond to user actions:

```javascript
// Message shown event
Gist.events.on('messageShown', (message) => {
  console.log(`Message shown: ${message.messageId}`);
  console.log(`Instance ID: ${message.instanceId}`);
});

// Message dismissed event
Gist.events.on('messageDismissed', (message) => {
  console.log(`Message dismissed: ${message.messageId}`);
});

// Message error event
Gist.events.on('messageError', (message) => {
  console.error(`Message error: ${message.messageId}`);
});

// Message action event (button clicks, etc.)
Gist.events.on('messageAction', (params) => {
  console.log(`Action: ${params.action}`);
  console.log(`Action name: ${params.name}`);
  console.log(`Message: ${params.message.messageId}`);
});
```

<Note>
  Events are emitted using an event emitter pattern. You can subscribe to multiple events and handle them as needed for analytics, navigation, or custom logic.
</Note>

## Complete Example

Here's a complete working example that puts it all together:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>Gist Web SDK Example</title>
  </head>
  <body>
    <div id="banner"></div>
    <button onclick="showWelcome()">Show Welcome Message</button>
    <button onclick="login()">Log In</button>
    <button onclick="logout()">Log Out</button>

    <script src="https://unpkg.com/customerio-gist-web/dist/gist.min.js"></script>
    <script>
      // Initialize SDK
      Gist.setup({
        siteId: 'your-site-id',
        dataCenter: 'us',
        logging: true,
        useAnonymousSession: true
      });

      // Set initial route
      Gist.setCurrentRoute('home');

      // Set up event listeners
      Gist.events.on('messageShown', (message) => {
        console.log(`Message shown: ${message.messageId}`);
      });

      Gist.events.on('messageDismissed', (message) => {
        console.log(`Message dismissed: ${message.messageId}`);
      });

      Gist.events.on('messageAction', (params) => {
        console.log(`Action: ${params.action} - ${params.name}`);
      });

      // Helper functions
      function showWelcome() {
        Gist.showMessage({
          messageId: 'welcome',
          position: 'center'
        });
      }

      function login() {
        Gist.setUserToken('user-123');
      }

      function logout() {
        Gist.clearUserToken();
      }
    </script>
  </body>
</html>
```

## Working with the Inbox

The SDK provides inbox functionality for persistent messages:

<Steps>
  <Step title="Get Inbox Messages">
    Retrieve all inbox messages for the current user:

    ```javascript
    const messages = await Gist.getInboxMessages();
    console.log(messages);
    ```
  </Step>

  <Step title="Get Unopened Count">
    Display a badge with the count of unread messages:

    ```javascript
    const count = await Gist.getInboxUnopenedCount();
    document.getElementById('badge').textContent = count;
    ```
  </Step>

  <Step title="Mark as Read/Unread">
    Update the read state of an inbox message:

    ```javascript
    await Gist.updateInboxMessageOpenState(queueId, true);  // Mark as read
    await Gist.updateInboxMessageOpenState(queueId, false); // Mark as unread
    ```
  </Step>

  <Step title="Delete Messages">
    Remove a message from the inbox:

    ```javascript
    await Gist.removeInboxMessage(queueId);
    ```
  </Step>

  <Step title="Listen to Inbox Updates">
    Subscribe to inbox update events:

    ```javascript
    Gist.events.on('messageInboxUpdated', async (messages) => {
      const count = await Gist.getInboxUnopenedCount();
      updateBadge(count);
      renderInboxList(messages);
    });
    ```
  </Step>
</Steps>

## Setting Custom Attributes

Set custom attributes on the current user for advanced targeting:

```javascript
// Set a custom attribute
Gist.setCustomAttribute('plan', 'premium');
Gist.setCustomAttribute('signup_date', '2024-01-15');

// Remove a specific attribute
Gist.removeCustomAttribute('plan');

// Clear all custom attributes
Gist.clearCustomAttributes();
```

## Setting User Locale

Set the user's locale for localized message content:

```javascript
Gist.setUserLocale('en-US');
Gist.setUserLocale('es-ES');
Gist.setUserLocale('fr-FR');
```
