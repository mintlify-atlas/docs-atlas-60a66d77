---
title: Custom Attributes
description: Set and manage custom user attributes for personalized messaging
---

Custom attributes allow you to store user-specific data that can be used to personalize messages and target specific user segments. The Gist Web SDK provides methods to set, retrieve, and manage these attributes.

## Overview

Custom attributes are:

- Stored locally in browser storage with a 30-day TTL
- Automatically included with every message request
- Available for use in message templates via Liquid syntax
- Persisted across sessions until manually cleared or expired
- Synchronized with message properties for personalization

## Setting Attributes

### Set a Single Attribute

Use `setCustomAttribute()` to store a custom attribute:

```
Gist.setCustomAttribute('plan', 'premium');
Gist.setCustomAttribute('userName', 'John Doe');
Gist.setCustomAttribute('accountAge', 365);
Gist.setCustomAttribute('preferences', { theme: 'dark', notifications: true });
```

### Supported Data Types

You can store any JSON-serializable value:

- Strings: `'premium'`, `'john@example.com'`
- Numbers: `42`, `3.14`, `100`
- Booleans: `true`, `false`
- Objects: `{ key: 'value' }`
- Arrays: `['tag1', 'tag2']`
- Null: `null`

```
// String
Gist.setCustomAttribute('email', 'user@example.com');

// Number
Gist.setCustomAttribute('loginCount', 15);

// Boolean
Gist.setCustomAttribute('isPremium', true);

// Object
Gist.setCustomAttribute('settings', {
  theme: 'dark',
  language: 'en',
  notifications: true
});

// Array
Gist.setCustomAttribute('interests', ['technology', 'design', 'marketing']);
```

## Retrieving Attributes

While custom attributes are automatically included in message requests, you can also retrieve them in your application code.

<Note>
  The SDK stores attributes in a Map internally. Attributes are automatically serialized when saved to local storage and included with message requests.
</Note>

## Removing Attributes

### Remove a Single Attribute

Remove a specific attribute by key:

```
Gist.removeCustomAttribute('userName');
```

The method returns `true` if the attribute existed and was removed, `false` otherwise:

```
const wasRemoved = Gist.removeCustomAttribute('planType');
if (wasRemoved) {
  console.log('Plan type attribute removed');
} else {
  console.log('Attribute did not exist');
}
```

### Clear All Attributes

Remove all custom attributes at once:

```
Gist.clearCustomAttributes();
console.log('All custom attributes cleared');
```

This is useful when:

- A user logs out
- You want to reset personalization
- Testing different attribute combinations

## Using Attributes in Messages

Custom attributes are automatically passed to message templates and can be referenced using Liquid syntax:

<Steps>
  <Step title="Set attributes in your app">
    ```javascript
    Gist.setCustomAttribute('userName', 'Sarah');
    Gist.setCustomAttribute('plan', 'premium');
    Gist.setCustomAttribute('trialDaysLeft', 7);
    ```
  </Step>
  
  <Step title="Reference in message template">
    In your Customer.io message template:
    
    ```liquid
    Hello {{ custom_attributes.userName }}!
    
    Your {{ custom_attributes.plan }} plan has {{ custom_attributes.trialDaysLeft }} days remaining.
    ```
  </Step>
  
  <Step title="Display the message">
    ```javascript
    await Gist.showMessage({
      messageId: 'trial-reminder'
    });
    ```
    
    The message will render with the custom attributes:
    
    "Hello Sarah! Your premium plan has 7 days remaining."
  </Step>
</Steps>

## Common Use Cases

### User Profile Information

Store user profile data for personalization:

```
function updateUserProfile(user) {
  Gist.setCustomAttribute('userName', user.name);
  Gist.setCustomAttribute('email', user.email);
  Gist.setCustomAttribute('accountType', user.accountType);
  Gist.setCustomAttribute('joinDate', user.joinDate);
  Gist.setCustomAttribute('avatarUrl', user.avatarUrl);
}

// When user logs in
updateUserProfile(currentUser);
```

### Subscription and Plan Information

```
function setSubscriptionInfo(subscription) {
  Gist.setCustomAttribute('plan', subscription.plan);
  Gist.setCustomAttribute('billingCycle', subscription.billingCycle);
  Gist.setCustomAttribute('renewalDate', subscription.renewalDate);
  Gist.setCustomAttribute('features', subscription.features);
}

setSubscriptionInfo({
  plan: 'Pro',
  billingCycle: 'monthly',
  renewalDate: '2024-04-01',
  features: ['unlimited-storage', 'priority-support', 'api-access']
});
```

### Usage Metrics

Track usage for targeted messaging:

```
function trackUsage() {
  Gist.setCustomAttribute('projectCount', userProjects.length);
  Gist.setCustomAttribute('storageUsed', calculateStorageUsed());
  Gist.setCustomAttribute('lastActiveDate', new Date().toISOString());
  Gist.setCustomAttribute('featureUsage', {
    export: 25,
    share: 10,
    collaborate: 5
  });
}

// Update usage metrics periodically
setInterval(trackUsage, 60000); // Every minute
```

### Preferences and Settings

```
function saveUserPreferences(preferences) {
  Gist.setCustomAttribute('language', preferences.language);
  Gist.setCustomAttribute('timezone', preferences.timezone);
  Gist.setCustomAttribute('emailNotifications', preferences.emailNotifications);
  Gist.setCustomAttribute('theme', preferences.theme);
}

saveUserPreferences({
  language: 'en',
  timezone: 'America/New_York',
  emailNotifications: true,
  theme: 'dark'
});
```

### A/B Testing Cohorts

```
// Assign user to test cohort
const cohort = Math.random() < 0.5 ? 'control' : 'variant';
Gist.setCustomAttribute('abTestCohort', cohort);
Gist.setCustomAttribute('experimentId', 'checkout-flow-v2');

// Show different messages based on cohort
if (cohort === 'variant') {
  await Gist.showMessage({ messageId: 'new-checkout-flow' });
} else {
  await Gist.showMessage({ messageId: 'standard-checkout' });
}
```

## Dynamic Attributes

Update attributes dynamically based on user behavior:

```
// Track page views
let pageViews = 0;
function trackPageView() {
  pageViews++;
  Gist.setCustomAttribute('pageViewCount', pageViews);
  
  // Show onboarding message after 3 page views
  if (pageViews === 3) {
    Gist.showMessage({ messageId: 'onboarding-tour' });
  }
}

// Track feature adoption
function trackFeatureUse(featureName) {
  const featureUsage = Gist.getCustomAttribute('featureUsage') || {};
  featureUsage[featureName] = (featureUsage[featureName] || 0) + 1;
  Gist.setCustomAttribute('featureUsage', featureUsage);
}

// Track engagement score
function calculateEngagementScore() {
  const score = calculateScore(); // Your scoring logic
  Gist.setCustomAttribute('engagementScore', score);
  
  // Show upgrade prompt for highly engaged users
  if (score > 80) {
    Gist.showMessage({ messageId: 'power-user-upgrade' });
  }
}
```

## Combining with Message Properties

Custom attributes work alongside message properties for maximum flexibility:

```
// Set persistent attributes
Gist.setCustomAttribute('userName', 'Alice');
Gist.setCustomAttribute('plan', 'free');

// Show message with additional context-specific properties
await Gist.showMessage({
  messageId: 'upgrade-prompt',
  properties: {
    // These are specific to this message instance
    upgradeOffer: '50% off',
    validUntil: '2024-03-31'
  }
});

// In your message template, you can use both:
// {{ custom_attributes.userName }} - from custom attributes
// {{ upgradeOffer }} - from message properties
```

## Session Management

### On User Login

```
function handleUserLogin(user) {
  // Clear any previous user's attributes
  Gist.clearCustomAttributes();
  
  // Set user token for authenticated session
  Gist.setUserToken(user.id);
  
  // Set new user's attributes
  Gist.setCustomAttribute('userName', user.name);
  Gist.setCustomAttribute('email', user.email);
  Gist.setCustomAttribute('plan', user.subscription.plan);
  Gist.setCustomAttribute('loginCount', user.loginCount);
}
```

### On User Logout

```
function handleUserLogout() {
  // Clear custom attributes
  Gist.clearCustomAttributes();
  
  // Clear user token
  Gist.clearUserToken();
  
  console.log('User session cleared');
}
```

## Storage and Persistence

Custom attributes are stored in browser local storage:

- **Storage Key**: `gist.web.customAttributes`
- **TTL**: 30 days from the last update
- **Size**: Limited by browser local storage quota (typically 5-10MB)
- **Serialization**: Automatically handled by the SDK

```
// Attributes persist across page reloads
Gist.setCustomAttribute('theme', 'dark');

// After page reload, the attribute is still available
// and will be included in message requests
```

## Validation and Error Handling

The SDK validates attribute keys:

```
// Valid keys
Gist.setCustomAttribute('userName', 'John'); // Returns true
Gist.setCustomAttribute('plan_type', 'premium'); // Returns true

// Invalid keys
Gist.setCustomAttribute('', 'value'); // Returns false, logs error
Gist.setCustomAttribute(null, 'value'); // Returns false, logs error
Gist.setCustomAttribute(123, 'value'); // Returns false, logs error
```

Always check return values when setting attributes:

```
const success = Gist.setCustomAttribute('userPlan', plan);
if (!success) {
  console.error('Failed to set custom attribute');
  // Handle the error appropriately
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Keep attribute names consistent">
    Use consistent naming conventions for attributes:
    
    ```javascript
    // Good: camelCase
    Gist.setCustomAttribute('userName', 'John');
    Gist.setCustomAttribute('accountType', 'premium');
    
    // Also good: snake_case
    Gist.setCustomAttribute('user_name', 'John');
    Gist.setCustomAttribute('account_type', 'premium');
    ```
  </Accordion>
  
  <Accordion title="Don't store sensitive data">
    Never store sensitive information in custom attributes:
    
    ```javascript
    // ❌ Don't do this
    Gist.setCustomAttribute('creditCard', '4111-1111-1111-1111');
    Gist.setCustomAttribute('password', 'user-password');
    Gist.setCustomAttribute('ssn', '123-45-6789');
    
    // ✅ Do this instead
    Gist.setCustomAttribute('hasPaymentMethod', true);
    Gist.setCustomAttribute('lastFourDigits', '1111');
    ```
  </Accordion>
  
  <Accordion title="Update attributes strategically">
    Balance between keeping attributes up-to-date and avoiding excessive storage writes:
    
    ```javascript
    // Update on significant changes
    Gist.setCustomAttribute('plan', newPlan); // Update immediately
    
    // Batch less critical updates
    let metrics = {};
    function updateMetrics() {
      metrics.pageViews++;
      // Save every 5 page views or on page unload
      if (metrics.pageViews % 5 === 0) {
        Gist.setCustomAttribute('pageViews', metrics.pageViews);
      }
    }
    ```
  </Accordion>
  
  <Accordion title="Clear attributes on logout">
    Always clear attributes when users log out to prevent data leakage:
    
    ```javascript
    function logout() {
      Gist.clearCustomAttributes();
      Gist.clearUserToken();
      // Additional logout logic
    }
    ```
  </Accordion>
</AccordionGroup>
