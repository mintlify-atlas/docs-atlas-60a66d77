## Overview

Messages are the core of the Gist Web SDK. The SDK automatically delivers messages from your Customer.io campaigns and provides APIs to manually show or embed messages.

## Message Types

The SDK supports three primary message display types:

### Overlay Messages

Overlay messages appear on top of your page content, typically as modals or full-screen takeovers.

```javascript
// Manually show an overlay message
const instanceId = await Gist.showMessage({
  messageId: 'msg_abc123'
});
```

Overlay messages:
- Block interaction with the underlying page
- Can be dismissed by clicking outside (if configured)
- Only one overlay message can be shown at a time
- Automatically scale to fit the viewport (if configured)

Source: `src/managers/message-manager.ts:54-84`

### Embedded Messages

Embedded messages are rendered inline within specific elements on your page.

```javascript
// Embed a message in a specific element
const instanceId = Gist.embedMessage(
  { messageId: 'msg_abc123' },
  'banner-container'
);
```

```html
<!-- Your page HTML -->
<div id="banner-container"></div>
```

Embedded messages:
- Render within the specified DOM element
- Multiple embedded messages can be shown simultaneously
- Height adjusts automatically to content (unless the element has a fixed height)
- Don't block interaction with other page elements

Source: `src/managers/message-manager.ts:86-117`

### Positioned Messages

Messages can be positioned at specific locations on the page using predefined positions:

```javascript
const message = {
  messageId: 'msg_abc123',
  properties: {
    gist: {
      position: 'top' // or 'bottom', 'center', etc.
    }
  }
};

await Gist.showMessage(message);
```

Available positions are managed by the page component manager and can be referenced by name.

Source: `src/managers/page-component-manager.ts`

## Message Structure

Messages are represented by the `GistMessage` interface:

```typescript
interface GistMessage {
  messageId: string;           // Required: Unique message identifier
  queueId?: string;            // Queue entry ID for tracked messages
  instanceId?: string;         // Runtime instance ID (generated by SDK)
  overlay?: boolean;           // Whether message is an overlay
  elementId?: string | null;   // Target element ID for embedded messages
  firstLoad?: boolean;         // Whether this is the first render
  shouldScale?: boolean;       // Whether to scale the message
  shouldResizeHeight?: boolean; // Whether to auto-resize height
  renderStartTime?: number;    // Timestamp when rendering started
  currentRoute?: string;       // Current route within the message
  position?: string | null;    // Position for positioned messages
  savedStepName?: string | null; // Saved step for multi-step messages
  isDisplayChange?: boolean;   // Whether this is a display change reload
  displaySettings?: DisplaySettings; // Message display configuration
  properties?: MessageProperties;    // Message properties from Customer.io
}
```

Source: `src/types.ts:13-29`

## Display Settings

Display settings control how messages appear on the page:

```typescript
interface DisplaySettings {
  displayType?: 'modal' | 'overlay' | 'inline' | 'tooltip';
  modalPosition?: string;       // Position for modal messages
  overlayPosition?: string;     // Position for overlay messages
  elementSelector?: string;     // CSS selector for inline messages
  maxWidth?: number;            // Maximum width in pixels
  overlayColor?: string;        // Overlay background color (rgba)
  dismissOutsideClick?: boolean; // Whether clicking outside dismisses
}
```

Display settings can be updated at runtime:

```javascript
const updated = Gist.updateMessageDisplaySettings(instanceId, {
  maxWidth: 600,
  overlayColor: 'rgba(0, 0, 0, 0.8)',
  dismissOutsideClick: true
});
```

Source: `src/types.ts:31-39`, `src/gist.js:164-172`

## Message Lifecycle

Messages go through a defined lifecycle from queue to display to dismissal:

### 1. Queue Phase

Messages enter the queue through either:
- **Automatic delivery**: Fetched from Customer.io based on user campaigns
- **Manual injection**: Added via `showMessage()` or `embedMessage()`

The queue manager checks for eligible messages and determines which should be displayed based on:
- Routing rules
- Priority ordering
- Existing displayed messages

Source: `src/managers/queue-manager.ts:68-82, 100-153`

### 2. Loading Phase

```javascript
// Loading starts
message.renderStartTime = new Date().getTime();
message.firstLoad = true;
message.instanceId = uuidv4(); // Generate unique instance ID
```

The SDK:
1. Generates a unique instance ID
2. Creates an iframe to load the message content
3. Passes message properties and custom attributes to the renderer
4. Starts tracking render time

Source: `src/managers/message-manager.ts:68-78, 102-112, 182-214`

### 3. Shown Phase

Once the message renders:

```javascript
Gist.events.on('messageShown', (message) => {
  console.log('Message shown:', message.messageId);
});
```

The SDK:
1. Displays the message (overlay or embedded)
2. Fires the `messageShown` event
3. Logs the message view to Customer.io (unless persistent)
4. Marks broadcast messages as seen locally

Source: `src/managers/message-manager.ts:266-286`, `src/gist.js:176-179`

### 4. Interaction Phase

Users can interact with message content:

```javascript
Gist.events.on('messageAction', ({ message, action, name }) => {
  console.log('User clicked:', action);
});
```

Actions can include:
- **Custom actions**: Links and buttons defined in the message
- **System actions**: Built-in actions like `gist://close` or `gist://showMessage`
- **Page navigation**: Deep links to other pages or external URLs

Source: `src/managers/message-manager.ts:288-349`, `src/gist.js:193-202`

### 5. Dismissal Phase

```javascript
// Manual dismissal
await Gist.dismissMessage(instanceId);

// Listen for dismissal
Gist.events.on('messageDismissed', (message) => {
  console.log('Message dismissed:', message.messageId);
});
```

The SDK:
1. Hides the message visually
2. Fires the `messageDismissed` event
3. Removes persistent message state
4. Logs broadcast dismissals locally
5. Checks the queue for the next message

Source: `src/gist.js:146-152, 181-186`, `src/managers/message-manager.ts:119-126`

## Persistent Messages

Persistent messages remain visible across page navigations and browser sessions:

```javascript
const message = {
  messageId: 'msg_abc123',
  properties: {
    gist: {
      persistent: true
    }
  }
};
```

Persistent messages:
- Don't log views immediately (logged only when dismissed)
- Save their state (current step, display settings) in local storage
- Restore their state on subsequent page loads
- Require explicit dismissal via the `gist://close` action

Source: `src/managers/message-manager.ts:136-150, 275-280`, `src/managers/queue-manager.ts:119-132`

## Broadcast Messages

Broadcast messages are delivered to all users (or user segments) and tracked locally:

- Marked as "seen" when displayed
- Marked as "dismissed" when closed
- Only shown once unless configured with "show always" behavior
- Don't require a queue entry

Source: `src/managers/message-broadcast-manager.ts`

## Multi-Step Messages

Messages can contain multiple steps (routes):

```javascript
Gist.events.on('messageAction', ({ message, action }) => {
  if (action === 'next-step') {
    // User advanced to the next step
    // SDK automatically saves step state for persistent messages
  }
});
```

When users navigate between steps:
1. The SDK saves the current step name
2. Display settings can change per step
3. If display settings change, the message reloads with the new settings
4. For persistent messages, the step state is saved and restored

Source: `src/managers/message-manager.ts:351-381, 432-467`

## Message Queue Priority

When multiple messages are eligible, they're displayed in priority order:

```javascript
const orderedMessages = allMessages.sort(
  (a, b) => a.priority - b.priority
);
```

Lower priority numbers are shown first. The SDK processes messages one at a time until it finds one that can be displayed.

Source: `src/managers/queue-manager.ts:74-82`
